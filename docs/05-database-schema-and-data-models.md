# 05. Database Schema & Data Models

This manual describes the data layouts and relationships in MongoDB and Qdrant.

## 1. MongoDB Collections & Schemas

### A. Users (`users` collection)
Stores profile metadata synced from Keycloak.
```json
{
  "_id": "ObjectId",
  "keycloak_id": "string (UUID)",
  "email": "string",
  "first_name": "string",
  "last_name": "string",
  "role": "string (tenant | landlord | vendor | admin)",
  "phone": "string",
  "created_at": "double (timestamp)"
}
```

### B. Properties (`properties` collection)
Represents physical real estate units.
```json
{
  "_id": "ObjectId",
  "landlord_id": "string (UUID)",
  "name": "string",
  "address": {
    "street": "string",
    "city": "string",
    "state": "string",
    "zip": "string"
  },
  "units": [
    {
      "unit_id": "string",
      "unit_number": "string",
      "tenant_id": "string (UUID)",
      "monthly_rent": "double"
    }
  ]
}
```

### C. Leases (`leases` collection)
Defines rental terms and references parsed S3 storage paths.
```json
{
  "_id": "ObjectId",
  "tenant_id": "string (UUID)",
  "unit_id": "string",
  "property_id": "ObjectId",
  "start_date": "string (YYYY-MM-DD)",
  "end_date": "string (YYYY-MM-DD)",
  "monthly_rent": "double",
  "security_deposit": "double",
  "minio_pdf_path": "string",
  "status": "string (active | draft | expired)"
}
```

### D. Maintenance Tickets (`maintenance_tickets` collection)
Coordinates maintenance work orders.
```json
{
  "_id": "ObjectId",
  "tenant_id": "string (UUID)",
  "property_id": "ObjectId",
  "unit_id": "string",
  "title": "string",
  "description": "string",
  "category": "string (plumbing | electrical | HVAC | structural | appliance)",
  "status": "string (OPEN | ASSIGNED | RESOLVED | CLOSED)",
  "priority": "string (low | medium | high | emergency)",
  "vendor_id": "string (UUID)",
  "media_urls": ["string"],
  "created_at": "double"
}
```

### E. Payments (`payments` collection)
Logs financial ledger items.
```json
{
  "_id": "ObjectId",
  "tenant_id": "string (UUID)",
  "amount": "double",
  "payment_method": "string (card | ach)",
  "status": "string (PENDING | COMPLETED | FAILED)",
  "transaction_id": "string",
  "timestamp": "double"
}
```

## 2. Qdrant Vector Collection

### Collection: `lease_documents`
* **Vector Size**: 384 dimensions (generated by `all-MiniLM-L6-v2` Sentence Transformer).
* **Distance Metric**: Cosine similarity.
* **Payload Fields**:
```json
{
  "tenant_id": "string (UUID)",
  "lease_id": "string",
  "chunk_id": "integer",
  "content": "string (raw text chunk from lease PDF)"
}
```
* **Payload Filtering Index**: Indexed on `tenant_id` for fast query partitioning.
